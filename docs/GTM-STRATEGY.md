## üí∞ UNIT ECONOMICS (SACCO-First Model)

### **CAC via SACCO Partnerships:**

```yaml
Scenario: Conservative
  - SACCO membership: 10,000 active
  - M-Recon adoption: 30% (3,000 users)
  - Premium conversion: 20% (600 Premium)
  - Partnership cost: KES 150,000 setup + KES 50,000/year maintenance
  - CAC per user: KES 200,000 √∑ 3,000 = KES 67 per user
  - CAC per Premium: KES 200,000 √∑ 600 = KES 333 per Premium

Scenario: Optimistic
  - SACCO membership: 10,000 active
  - M-Recon adoption: 50% (5,000 users)
  - Premium conversion: 30% (1,500 Premium)
  - Partnership cost: KES 150,000 setup + KES 50,000/year maintenance
  - CAC per user: KES 200,000 √∑ 5,000 = KES 40 per user
  - CAC per Premium: KES 200,000 √∑ 1,500 = KES 133 per Premium
```

### **LTV (Customer Lifetime Value):**

```yaml
Premium User:
  - MRR: KES 499/month
  - Avg Lifetime: 18 months (conservative)
  - LTV: KES 499 √ó 18 = KES 8,982

LTV:CAC Ratios:
  - Conservative: KES 8,982 √∑ KES 333 = 27:1 ‚úÖ
  - Optimistic: KES 8,982 √∑ KES 133 = 68:1 ‚úÖ‚úÖ
  
Industry Benchmark: 3:1 is good, 5:1 is great
M-Recon Target: 20:1 (EXCEPTIONAL)
```

### **Break-Even Analysis:**

```yaml
Fixed Costs (Monthly):
  - Server (Supabase Pro): KES 8,000
  - Email (Resend): KES 3,000
  - Domain + SSL: KES 500
  - Mo salary (partial): KES 50,000
  Total: KES 61,500/month

Break-Even:
  - Premium users needed: 61,500 √∑ 499 = 124 users
  - OR: 1 pilot SACCO with 30% adoption + 20% Premium conversion
  - = 10,000 √ó 30% √ó 20% = 600 Premium users (4.8x break-even!)
```

## üö® RISK MITIGATION

### **Risk 1: SACCO Adoption Lower Than Expected (30% vs 50%)**

```yaml
Mitigation:
  - Offer 3-month free Premium for early adopters
  - In-person demos at SACCO branches
  - Train SACCO loan officers on M-Recon
  
Worst Case:
  - 30% adoption = 3,000 users per SACCO
  - Still achieves break-even with 1 SACCO
```

### **Risk 2: PDF Parsing Accuracy <85%**

```yaml
Mitigation:
  - PoC BEFORE MVP commitment (50 PDFs)
  - Manual review option (human-in-loop)
  - Transparent accuracy score shown to users
  
Worst Case:
  - If <70% accuracy ‚Üí Pivot to Daraja API (formal businesses only)
  - Smaller TAM but higher accuracy
```

### **Risk 3: WEF Bank Account Requirement Can't Be Solved**

```yaml
Mitigation:
  - Focus on SACCOs (don't need bank accounts)
  - Partner with digital banks (KCB, Equity) for easy account opening
  - Position M-Recon as "bank account prep tool"
  
Worst Case:
  - WEF remains inaccessible ‚Üí OK! SACCOs are primary anyway
```

### **Risk 4: Premium Conversion Lower Than 20%**

```yaml
Mitigation:
  - Free tier with 50 transactions (hook users)
  - Loan referral bonus (free month if approved)
  - SACCO co-marketing (subsidized Premium)
  
Worst Case:
  - 10% conversion = still 500 Premium users per SACCO @ 50% adoption
  - LTV:CAC still strong (13:1)
```
